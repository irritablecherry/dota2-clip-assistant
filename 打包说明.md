# 📦 打包说明 - Dota2 切片助手

## 快速打包

### 方法一：一键打包（推荐）

双击运行 `一键打包.bat`，根据提示选择打包模式即可。

### 方法二：手动打包

1. **安装依赖**
   ```bash
   pip install -r requirements.txt
   pip install pyinstaller
   ```

2. **运行打包脚本**
   ```bash
   python build.py
   ```

3. **使用 spec 文件打包**
   ```bash
   pyinstaller dota2_clip_assistant.spec
   ```

---

## 打包模式选择

### 单文件模式 (--onefile)
- ✅ 优点：只有一个 exe 文件，便于分发
- ❌ 缺点：体积较大（约 200MB+），启动较慢
- 📁 输出位置：`dist/dota2_clip_assistant.exe`

### 目录模式 (--onedir)
- ✅ 优点：体积较小，启动较快
- ❌ 缺点：文件较多，需要整个目录一起分发
- 📁 输出位置：`dist/dota2_clip_assistant/dota2_clip_assistant.exe`

**推荐使用目录模式**，除非你需要单个文件便于传播。

## 减小打包体积的方法

1. **排除不必要的模块**
   - 已在 `build.py` 中配置了 exclude 列表
   - 排除 matplotlib、scipy、tensorflow 等大型库

2. **使用 UPX 压缩**
   - 已在 spec 文件中启用 `upx=True`
   - 可减小 30-50% 体积

3. **移除不必要的数据文件**
   - 只打包必需的模型和配置文件
   - 不打包视频、图片等用户文件

## 常见问题

### Q1: 打包后运行报错 "找不到模型文件"
**解决：** 确保 model 目录被正确打包到 exe 同级目录

### Q2: 打包后运行报错 "缺少 DLL 文件"
**解决：** 
- 检查 `hiddenimports` 是否包含所有必要模块
- 查看错误日志，添加缺失的模块名

### Q3: 打包体积太大
**解决：**
- 使用目录模式代替单文件模式
- 检查是否打包了不必要的大文件
- 启用 UPX 压缩

### Q4: 打包后运行闪退
**解决：**
- 运行 `dota2_clip_assistant.exe --debug` 查看错误
- 检查是否缺少 Visual C++ 运行库
- 确保 Python 版本 >= 3.8
